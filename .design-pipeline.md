# Design Pipeline

Three tools, three roles, one source of truth (`DESIGN.md`). Each tool operates at a different phase — they do not overlap.

```
DESIGN.md ─────────── source of truth (tokens: colors, fonts, spacing, components)
     │                 Single file, symlinked to .stitch/DESIGN.md
     │
     ├── Stitch ────── mockup generator (layout: where things go on the page)
     │                 Used during /spec_kit:plan for new page/screen tasks ONLY
     │
     ├── shadcn/ui ─── component library + charts (code: actual React components)
     │                 Used during /spec_kit:complete by implementing agents
     │
     └── impeccable ── quality gate (verification: is the result good enough?)
                       Used after implementation. /audit scores 0-20, must pass ≥14
```

---

## Full Pipeline

```
/spec_kit:plan
  ├── /shape         ← design brief (UX goals, constraints, strategy)
  ├── enhance-prompt ← refine prompt with DESIGN.md tokens
  └── stitch-design  ← generate mockup matching Linear style
        ↓ save HTML/PNG to specs/{folder}/scratch/

/spec_kit:complete
  └── agents build   ← read design brief + mockup + DESIGN.md → shadcn/ui code
        ↓

Verify
  ├── /critique      ← UX review (persona-based, emotional resonance)
  └── /audit         ← technical review (a11y, perf, anti-patterns) → ≥14/20
        ↓ fail?
      /audit recommends fix commands → /typeset, /layout, /colorize, etc.
        ↓
      /polish         ← final pass (alignment, spacing, states)
        ↓
      re-audit        ← repeat until pass
```

## 1. DESIGN.md — Source of Truth

- **Files**: `DESIGN.md` (root) ← symlink → `.stitch/DESIGN.md` (same file)
- **Origin**: Linear-inspired, installed via [getdesign.md](https://getdesign.md/), 367 lines
- **Context**: `.impeccable.md` defines brand personality, audience, aesthetic direction
- **Contains**: Color palette, typography (Inter Variable cv01+ss03), spacing scale, component styles, depth/elevation, responsive breakpoints, do's and don'ts, agent prompt guide
- **Non-negotiables**: Dark-mode-native `#08090a`, Inter Variable `"cv01","ss03"`, weight 510, brand indigo `#5e6ad2`/`#7170ff` CTAs only, `#f7f8f8` text (no pure white), semi-transparent white borders

## 2. Stitch — Mockup Generator (Plan Phase)

### When to use
- During `/spec_kit:plan`, when a task is a **NEW page/screen** with non-standard layout
- Examples: analytics dashboard, content calendar, trend dashboard, scoring display

### When to skip
- Standard patterns: CRUD form, data table, settings page → shadcn/ui has these built-in
- Stitch MCP disabled or API key not set
- Small components (button, card, modal)

### Skills
1. **`/shape`** — Run first. Structured discovery interview → design brief (UX goals, constraints, responsive strategy). Output guides both Stitch and implementing agents.
2. **`enhance-prompt`** — Reads `DESIGN.md`, injects Linear tokens into prompt, structures page description into sections.
3. **`stitch-design`** — Orchestrator: reads `.stitch/DESIGN.md` → calls `generate_screen_from_text` → saves HTML/PNG to `specs/{folder}/scratch/`.

### Config
- **MCP**: `stitch` in `.mcp.json` — `STITCH_API_KEY` in `.env`
- **Limits**: 350 generations/month, minutes per generation
- **Output**: HTML/Tailwind (not Next.js) — layout reference only, not code source

### Rules
- DESIGN.md is source of truth — when mockup conflicts with tokens, follow DESIGN.md
- NOT used for code generation, design system creation, or component conversion

## 3. impeccable — Quality Gate (Complete Phase)

### Mandatory checks (automated via FRONTEND DESIGN RULE)
- **`/audit`** — Technical quality: accessibility, performance, theming, responsive, anti-patterns. Score 0-20, must pass ≥14/20 with Anti-Patterns ≥3/4.
- **`/critique`** — UX quality: visual hierarchy, information architecture, emotional resonance, cognitive load. Persona-based testing.

### Fix commands (triggered by /audit recommendations)

| Command | Fixes |
|---------|-------|
| `/polish` | Alignment, spacing, interaction states, micro-details |
| `/typeset` | Font choices, hierarchy, sizing, weight, readability |
| `/layout` | Grid, spacing rhythm, visual hierarchy, composition |
| `/colorize` | Color strategy, contrast, monotone issues |
| `/adapt` | Responsive breakpoints, touch targets, viewport adaptation |
| `/optimize` | Performance, bundle size, animation jank |
| `/clarify` | UX copy, error messages, labels, instructions |
| `/animate` | Transitions, micro-interactions, motion effects |

### Flow
```
/audit → score < 14 or anti-patterns < 3?
  YES → read audit report → run recommended commands → /polish → re-audit
  NO  → PASS → claim done
```

### On-demand skills (not in pipeline — use manually)
| Command | When |
|---------|------|
| `/bolder` | Design looks bland, needs more visual impact |
| `/quieter` | Design too aggressive, needs calming |
| `/delight` | Add personality, micro-interactions, joy |
| `/overdrive` | Go all-out: shaders, spring physics, scroll-driven effects |
| `/distill` | Remove complexity, declutter |

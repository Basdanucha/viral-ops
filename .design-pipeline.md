## 1. DESIGN.md ‚Äî Source of Truth

- **Files**: `DESIGN.md` (root) ‚Üê symlink ‚Üí `.stitch/DESIGN.md` (same file)
- **Origin**: Linear-inspired, installed via [getdesign.md](https://getdesign.md/), 367 lines
- **Context**: `.impeccable.md` defines brand personality, audience, aesthetic direction
- **Contains**: Color palette, typography (Inter Variable cv01+ss03), spacing scale, component styles, depth/elevation, responsive breakpoints, do's and don'ts, agent prompt guide
- **Non-negotiables**: Dark-mode-native `#08090a`, Inter Variable `"cv01","ss03"`, weight 510, brand indigo `#5e6ad2`/`#7170ff` CTAs only, `#f7f8f8` text (no pure white), semi-transparent white borders

---

## 2. Stitch ‚Äî Mockup Generator (Step 10 Development)

### When to use
- During **Step 10 (Development)** of `/spec_kit:complete`, before writing code for a **NEW page/screen** with non-standard layout
- Examples: analytics dashboard, content calendar, trend dashboard, scoring display

### When to skip
- Standard patterns: CRUD form, data table, settings page ‚Üí shadcn/ui has these built-in
- Stitch MCP disabled or API key not set
- Small components (button, card, modal)

### Skills (mapped to `/spec_kit:complete` steps)
1. **`/shape`** ‚Äî Run during **Step 6 (Planning)**. Structured discovery interview ‚Üí design brief (UX goals, constraints, responsive strategy). Include in plan.md.
2. **`enhance-prompt`** ‚Äî Run during **Step 10**, before writing UI code. Reads `DESIGN.md`, injects Linear tokens into prompt.
3. **`stitch-design`** ‚Äî Run during **Step 10**, after enhance-prompt. Reads `.stitch/DESIGN.md` ‚Üí calls `generate_screen_from_text` ‚Üí agent uses mockup as layout reference (not saved separately).

### Config
- **MCP**: `stitch` in `.mcp.json` ‚Äî `STITCH_API_KEY` in `.env`
- **Limits**: 350 generations/month, minutes per generation
- **Output**: HTML/Tailwind (not Next.js) ‚Äî layout reference only, not code source

### Rules
- DESIGN.md is source of truth ‚Äî when mockup conflicts with tokens, follow DESIGN.md
- NOT used for code generation, design system creation, or component conversion

## 3. impeccable ‚Äî Quality Gate (Complete Phase)

### Mandatory checks (automated via FRONTEND DESIGN RULE)
- **`/audit`** ‚Äî Technical quality: accessibility, performance, theming, responsive, anti-patterns. Score 0-20, must pass ‚â•14/20 with Anti-Patterns ‚â•3/4.
- **`/critique`** ‚Äî UX quality: visual hierarchy, information architecture, emotional resonance, cognitive load. Persona-based testing.

### Fix commands (triggered by /audit recommendations)

| Command | Fixes |
|---------|-------|
| `/polish` | Alignment, spacing, interaction states, micro-details |
| `/typeset` | Font choices, hierarchy, sizing, weight, readability |
| `/layout` | Grid, spacing rhythm, visual hierarchy, composition |
| `/colorize` | Color strategy, contrast, monotone issues |
| `/adapt` | Responsive breakpoints, touch targets, viewport adaptation |
| `/optimize` | Performance, bundle size, animation jank |
| `/clarify` | UX copy, error messages, labels, instructions |
| `/animate` | Transitions, micro-interactions, motion effects |

### Flow
```
/audit ‚Üí score < 14 or anti-patterns < 3?
  YES ‚Üí read audit report ‚Üí run recommended commands ‚Üí /polish ‚Üí re-audit
  NO  ‚Üí PASS ‚Üí claim done
```

### On-demand skills (not in pipeline ‚Äî use manually)
| Command | When |
|---------|------|
| `/bolder` | Design looks bland, needs more visual impact |
| `/quieter` | Design too aggressive, needs calming |
| `/delight` | Add personality, micro-interactions, joy |
| `/overdrive` | Go all-out: shaders, spring physics, scroll-driven effects |
| `/distill` | Remove complexity, declutter |
